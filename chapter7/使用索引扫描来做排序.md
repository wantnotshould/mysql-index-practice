# 使用索引扫描来做排序

MySQL 有两种方式可以生成有序的结果：
- 通过排序操作
- 按索引顺序扫描

如果在 `EXPLAIN` 的输出结果中，type 列的值为 “index”，则说明 MySQL 使用了索引扫描来做排序（注释，不要和 Extra 列的 “Using index” 搞混）

## 准备环境

```sql
CREATE TABLE products (
    id INT AUTO_INCREMENT PRIMARY KEY,
    category_id INT NOT NULL,
    price DECIMAL(10, 2) NOT NULL,
    created_at DATETIME NOT NULL,
    product_name VARCHAR(100),
    -- 核心：复合索引 (category_id, price, created_at)
    INDEX idx_cat_price_time (category_id, price, created_at)
) ENGINE=INNODB;

INSERT INTO products (category_id, price, created_at, product_name) VALUES
(1, 100.00, '2023-01-01 10:00:00', 'Phone A'),
(1, 200.00, '2023-01-02 11:00:00', 'Phone B'),
(1, 200.00, '2023-01-03 12:00:00', 'Phone C'),
(2, 50.00,  '2023-01-01 09:00:00', 'Case X'),
(2, 150.00, '2023-01-04 15:00:00', 'Case Y');
```

### 索引扫描排序

```sql
-- 场景：查询分类为 1 的产品，并按价格升序排列
EXPLAIN SELECT category_id, price, created_at FROM products  WHERE category_id = 1  ORDER BY price\G
*************************** 1. row ***************************
           id: 1
  select_type: SIMPLE
        table: products
   partitions: NULL
         type: ref
possible_keys: idx_cat_price_time
          key: idx_cat_price_time
      key_len: 4
          ref: const
         rows: 3
     filtered: 100.00
        Extra: Using index
1 row in set, 1 warning (0.00 sec)
```

由于索引是先按 `category_id` 排序，在相同的 `category_id` 下按 `price` 排序。既然我们锁定了 `category_id = 1`，那么接下来的索引记录天然就是按 `price` 有序的

### 没有利用索引扫描排序

```sql
-- 场景：按价格排序，但没有指定 category_id，或者排序方向不一致
EXPLAIN SELECT category_id, price, created_at FROM products  ORDER BY price ASC, created_at DESC\G
*************************** 1. row ***************************
           id: 1
  select_type: SIMPLE
        table: products
   partitions: NULL
         type: index
possible_keys: NULL
          key: idx_cat_price_time
      key_len: 14
          ref: NULL
         rows: 5
     filtered: 100.00
        Extra: Using index; Using filesort
1 row in set, 1 warning (0.00 sec)
```

出现 `Using filesort`，索引是按 (`price ASC`, `created_at ASC`) 存储的。如果要求 `created_at` 倒序，或者跳过第一列直接排第二列，索引的顺序就失效了，MySQL 必须在内存中重新排序
